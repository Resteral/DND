# dnd-3d-vtt

Lightweight 3D virtual tabletop prototype focused on D&D 5e. Browser-first React + react-three-fiber front-end with optional real-time backends (local WebSocket for dev, Supabase for production). Designed to support 3D GLTF maps, token movement, and integrations (Foundry/Roll20 imports).

**Status:** scaffolded demo — token movement + local WS broadcast, 3D scene and UI overlay.

## Features (prototype)
- 3D map rendering with `react-three-fiber`
- Floating character tokens with overhead HP UI
- Click-to-move selected token; movement broadcasts over a local WebSocket server
- Starter server for local dev (Express + `ws`)

## Prerequisites
- Node.js 18+ and npm

## Local development
1. Install dependencies

```bash
npm install
```

2. Start the optional local server (used by the demo WebSocket client)

```bash
# opens port 3001
node server/index.js
```

3. Run the frontend dev server

```bash
npm run dev
```

Open `http://localhost:5173` (Vite default) and optionally ensure the local server is running on port 3001.

## Environment & Production (Vercel + Supabase)

Recommended production stack:
- Frontend: Vercel (deploy the Vite build)
- Backend: Supabase (Postgres + Auth + Storage + Realtime)

Environment variables for Vercel (frontend):

- `VITE_SUPABASE_URL` — your Supabase project URL
- `VITE_SUPABASE_ANON_KEY` — anon public key (safe for client)

Server-only secrets (do NOT expose to client):

- `SUPABASE_SERVICE_ROLE_KEY` — service role key for server operations (store in Vercel server envs only)

### Supabase integration (notes)

1. Install the supabase client for the frontend:

```bash
npm install @supabase/supabase-js
```

2. Example client initializer `src/lib/supabase.js`:

```js
import { createClient } from '@supabase/supabase-js';
export const supabase = createClient(import.meta.env.VITE_SUPABASE_URL, import.meta.env.VITE_SUPABASE_ANON_KEY);
```

3. Recommended `tokens` table (run in Supabase SQL editor):

```sql
create table public.tokens (
  id uuid primary key default gen_random_uuid(),
  map_id uuid not null,
  actor_id uuid,
  owner_id uuid,
  position jsonb not null,
  size numeric default 1,
  sight jsonb,
  metadata jsonb,
  updated_at timestamptz default now()
);
create index on public.tokens (map_id);
```

4. Realtime flow (production):
- Clients perform authenticated `UPDATE tokens SET position = ...` (or call an RPC) when moving tokens.
- Supabase Realtime broadcasts DB changes to subscribed clients.
- Enforce Row Level Security (RLS) policies so only owners or DMs can update relevant rows.

## Switching from local WS to Supabase Realtime

- Replace the local `src/lib/ws.js` usage with `supabase.channel()` subscription to `public.tokens` changes.
- Route authoritative updates through an RPC or serverless function to validate permissions before writing.
